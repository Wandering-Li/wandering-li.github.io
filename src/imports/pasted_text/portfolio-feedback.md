五、Featured Projects：这是你最强的部分，但仍然有可优化点

这一块是你目前最有竞争力的部分。
但我会给你几条直说的建议。

项目区优点
1. 结构正确

标题 → 简述 → tech stack → workflow → 大图 → 文字说明
这个顺序是合理的。

2. 读者能看懂项目

至少从截图看，招聘方能快速明白：

这是做什么的

流程大概怎么走

用了什么技术

你做了哪些模块

3. 三个项目彼此区分度不错

不会都像一个模子刻出来。

项目区问题
1. AI Search 的图片不够贴题

那张黑底球状网络图挺酷，但和项目本身关系不够直接。
它更像一个抽象科技图，不像产品/系统图。

你的 Building Assessment 那块，workflow 本身已经很贴题了。
但 AI Search 那张图没有同样的说服力。

2. D-TA 项目的大图有点像 stock/dashboard 图

这张图本身观感还行，但也有点“素材感”。
如果这个项目真有 UI 截图、架构图、窗口布局图，哪怕不那么美观，也会更真实。

3. Problem / Solution / What I Built / Impact 太整齐，略模板化

结构没问题，但语言上稍微有点“标准答案感”。

比如很多人看这种内容，会觉得：

这是你真实总结的吗

还是 AI 帮你写得很顺

这在 portfolio 里要小心，因为你投的是 AI 岗，别人会自然怀疑内容有多少是你自己的。

项目区最重要的改法
对每个项目都加一个更“像你本人”的模块

例如：

Engineering decisions

Why this architecture

Key challenges

Trade-offs

这会一下子把页面从“整理过的项目展示”变成“真正的工程 portfolio”。

比如 Building Assessment 项目可以加：

Key Engineering Decisions

Separated extraction from report generation to improve control and debugging

Used retrieval grounding to reduce hallucination in compliance-heavy sections

Structured the workflow into stages instead of relying on one-shot prompting

这种东西特别值钱。
因为这不是套模板能轻易写好的内容，而是工程判断。

六、内容真实性 / 可信度问题：有几处得小心

这块我得直接提醒你。

1. Impact 数字别乱写

比如：

reducing query time by 80%

sub-100ms latency

reduced report generation time from hours to minutes

这些数字如果你没有真实证据支撑，就危险。

portfolio 和简历一样，最怕这种：
看起来很厉害，但一问细节答不出来。

如果这些是粗估、原型环境观察、或者只是表达效果，不如换成更稳的说法。

例如：

significantly reduced manual report-writing effort

enabled faster query workflows for non-technical users

supported low-latency updates for real-time monitoring

更稳，也更不容易翻车。

2. Azure OpenAI 那个 tag 要确认

你前面不是说 Axon 主要是官方 API / AWS 吗？
这里 AI Search 项目 tag 里有 Azure OpenAI，如果这不是确实用过的，就不要放。

因为 tag 是强信号。
放了就默认你真的在这项目里用过。

3. C++ 这个点要一致

D-TA 项目主描述里是 Electron + React + Julia + ZeroMQ。
tag 里有 C++。如果只是背景技能，不是项目关键技术，最好别放在项目 tag 里。

七、Experience / Education / Footer：合格，但还有增强空间
Experience

简洁、干净、没问题。
但现在略弱，因为它只是“时间线卡片”，没有一点点 summary。

你可以给每条 experience 加一句 1 行的 very short descriptor，例如：

Built an AI-powered search prototype for internal product validation

Developed a real-time visualization system with a ZeroMQ-based pipeline

这样信息量会更高。

Education

干净，够用。
不过硕士这里可以考虑加一句很轻的补充，比如：

University of Ottawa, GPA 90/100
如果你想强化学术质量的话。

Footer

整体不错，CTA 明显。
但 “Let’s Connect” 这段挺标准，也稍微有点泛。

你可以再偏工程一点，例如：

Open to AI Engineer and Applied AI opportunities in backend, product, and intelligent systems.
这会更贴你的定位。
